From Parallel to Sequential: Introduction to Boosting

Ensemble Methods Recap:

  • Bagging/Random Forest: Build trees in parallel, average predictions
  • Goal: Reduce variance by averaging uncorrelated models

Boosting: A Different Approach

  • Build trees sequentially, each learning from previous mistakes
  • Goal: Reduce bias by focusing on difficult observations
  • Key insight: Slow learning often performs better

Boosting Philosophy:

“A procedure that combines many weak learners to create a strong learner”

Todays Focus: Understanding boosting algorithms

The Boosting Intuition

Traditional approach: Fit one complex model to all data

Boosting approach: Fit many simple models, each fixing previous errors

Analogy: Learning to ride a bike

  • Trial 1: Fall to the right Focus on right balance
  • Trial 2: Overcorrect, fall to left Focus on left balance
  • Trial 3: Combine lessons Better balance
  • Continue: Each attempt focuses on remaining weaknesses

In Machine Learning:

  1. Fit simple model (e.g., decision stump)
  2. Identify misclassified observations
  3. Give extra weight to these “difficult” cases
  4. Fit new model focusing on difficult cases
  5. Combine all models with weighted voting

Boosting Algorithm for Regression

Gradient Boosting Algorithm:

  1. Initialize: \(\hat{f}(x) = 0\), \(r_i = y_i\) for all \(i\)

  2. For \(b = 1, 2, \ldots, B\):

    • Fit tree: \(\hat{f}^b\) with \(d\) splits to training data \((X, r)\)
    • Update model: \(\hat{f}(x) \leftarrow \hat{f}(x) + \lambda \hat{f}^b(x)\)
    • Update residuals: \(r_i \leftarrow r_i - \lambda \hat{f}^b(x_i)\)

  3. Output: \(\hat{f}(x) = \sum_{b=1}^{B} \lambda \hat{f}^b(x)\)

Key Parameters:

  • \(B\): Number of trees (can overfit if too large!)
  • \(\lambda\): Shrinkage parameter (learning rate, typically 0.001-0.1)
  • \(d\): Tree depth (interaction depth, often \(d=1\) for stumps)

Boosting Step 1: Fit First Tree to Data

Whats happening:

  • Tree makes a simple split and predicts the mean in each region
  • We multiply by \(\lambda = 0.3\) (shrinkage) - only use 30% of the prediction
  • This is “slow learning” - intentionally conservative to avoid overfitting

Boosting Step 2: Calculate Residuals (Errors)

Key Insight:

  • Instead of fitting to y again, Tree 2 will target these residuals
  • Residuals = what our current model f(x) is getting wrong
  • This is the core of boosting: each tree corrects previous mistakes!

Boosting Step 3: Fit Tree 2 to Residuals

Key Point:

  • Tree 2 does NOT fit to y - it fits to the residuals!
  • Split location is different from Tree 1 (it targets where errors are large)
  • Again multiply by \(\lambda = 0.3\) before adding to our model

Boosting Step 4: Combined Model Improves!

\[f(x) = \lambda \cdot \text{Tree}_1(x) + \lambda \cdot \text{Tree}_2(x) + \lambda \cdot \text{Tree}_3(x) + \ldots\]

  • Each tree adds a correction to fix remaining errors
  • Continue for B iterations (typically 100-5000 trees)
  • \(\lambda\) controls how fast we learn (smaller = slower but more robust)

Why Fit to Residuals? Summary

  1. Tree 1 fits to y captures main patterns
  2. Tree 2 fits to \(r_1\) captures what Tree 1 missed
  3. Tree 3 fits to \(r_2\) captures what Trees 1+2 missed
  4. Continue for B iterations

Why This Works:

  • Each tree specializes in correcting specific errors
  • \(\lambda\) (shrinkage) prevents overfitting by taking small steps
  • Sequential learning builds complex patterns from simple pieces
  • Different splits at each iteration target different problem areas

Boosting in Practice: Boston Housing

# Load Boston data
set.seed(1)
train_idx <- sample(nrow(Boston), nrow(Boston)/2)
boston_train <- Boston[train_idx, ]
boston_test <- Boston[-train_idx, ]

# Fit boosted regression tree
boost_boston <- gbm(medv ~ ., data = boston_train,
                   distribution = "gaussian",  # For regression
                   n.trees = 5000,           # Number of trees
                   interaction.depth = 4,     # Tree depth
                   shrinkage = 0.01)         # Learning rate

# Make predictions
boost_pred <- predict(boost_boston, boston_test, n.trees = 5000)
boost_mse <- mean((boost_pred - boston_test$medv)^2)

Variable Importance: Boosting vs Random Forest

Random Forest Variable Importance:

  • Mean Decrease in Gini/Impurity: Total reduction in node impurity from splits on that variable, averaged across all trees
  • Mean Decrease in Accuracy: How much accuracy drops when the variable is randomly permuted (if importance=TRUE)

Gradient Boosting Variable Importance:

  • Relative Influence: Sum of squared improvements in the loss function from splits on that variable across all trees
  • Measures how much each variable contributes to reducing the overall prediction error
  • Weighted by the number of observations at each split

Key Difference:

  • Random Forest: Based on Gini impurity reduction or permutation
  • Gradient Boosting: Based on loss function improvement (residual reduction)
  • Both aim to quantify “how useful is this variable,” but calculate it differently
  • In practice, they often (but not always) agree on which variables are most important

Tuning Boosting Parameters

The Tuning Process:

  • Shrinkage (): Controls the learning rate - how much each tree contributes
    • Smaller values (0.001-0.01) require more trees but often perform better
    • Larger values (0.1-0.2) learn faster but may overfit
  • Number of Trees (B): Total iterations of boosting
    • Too few: underfitting (high bias)
    • Too many: overfitting (high variance)
  • Strategy: For each shrinkage value, test multiple tree counts to find the optimal combination
  • Trade-off: Smaller shrinkage needs more trees; larger shrinkage needs fewer trees

Interaction Depth: Controlling Model Complexity

  • Too shallow (depth = 1-2): May miss important patterns
  • Just right (depth = 3-5): Balances complexity and generalization
  • Too deep (depth = 6+): Overfits to training data noise

Tuning parameters with Cross-Validation

  • No OOB error: Unlike Random Forest, boosting doesnt use bootstrap sampling
  • Sequential training: Each tree uses all training observations
  • Solution: Must use cross-validation to estimate test error and tune parameters

Use the cv.folds option in gbm to accomplish this

  • cv.folds in gbm only tunes the number of trees (not shrinkage or interaction.depth)
  • As boosting builds trees sequentially (1, 2, 3, , n.trees), it calculates CV error at each iteration
  • Helps find the optimal stopping point to prevent overfitting
  • Other parameters (shrinkage, interaction.depth) must be tuned manually with multiple model fits
# Fit with 5-fold cross-validation
boost_cv <- gbm(medv ~ ., data = boston_train,
               distribution = "gaussian",
               n.trees = 5000,
               interaction.depth = 4,     # Fixed - not tuned by CV
               shrinkage = 0.01,          # Fixed - not tuned by CV
               cv.folds = 5)              # Enables CV for finding optimal trees
# Find optimal number of trees using CV error
best_iter <- gbm.perf(boost_cv, method = "cv")
# Make predictions using optimal number of trees
pred_cv <- predict(boost_cv, boston_test, n.trees = best_iter)

Best Practices for Boosting

Parameter Tuning Guidelines:

  • Start with: shrinkage = 0.01, interaction.depth = 4, n.trees = 3000
  • Cross-validation: Use cv.folds in gbm() for automatic tuning
  • Monitor: Training vs validation error to detect overfitting
  • Shrinkage: Smaller values (0.001-0.01) often work better with more trees

Common Mistakes:

  • Using too few trees (< 1000)
  • Setting shrinkage too high (> 0.1)
  • Ignoring interaction depth effects
  • Not monitoring for overfitting

Computational Considerations:

  • Boosting is sequential (cant parallelize easily)
  • Random Forest is parallel (faster on multi-core systems)
  • For large datasets, consider XGBoost or LightGBM
