Recap: Linear vs Logistic Regression

Linear Regression Problems for Classification:

  • Predictions can be outside [0,1] range
  • Assumes linear relationship between predictors and probability
  • Residuals don’t follow normal distribution for binary outcomes

Logistic Regression Solution:

  • Uses logistic (sigmoid) function to map any real number to [0,1]
  • Models log-odds (logit) as linear combination of predictors
  • Provides probabilities that can be converted to classifications

The Logistic Function

The logistic function transforms any real number to a probability between 0 and 1:

\[p(X) = \frac{e^{\beta_0 + \beta_1 X}}{1 + e^{\beta_0 + \beta_1 X}}\]

From Probability to Odds

Odds: \(\text{Odds} = \frac{p}{1-p} = e^{\beta_0 + \beta_1 X} = e^{\beta_0} e^{\beta_1 X}\)

  • A 1 unit increase in \(X\) multiplies the odds by \(e^{\beta_1}\)

Log-Odds (Logit): \(\text{logit}(p) = \log\left(\frac{p}{1-p}\right) = \beta_0 + \beta_1 X\)

  • A 1 unit increase in \(X\) increases log-odds by \(\beta_1\)

Example:

  Probability Odds Log_Odds
1         0.1 0.11    -2.20
2         0.3 0.43    -0.85
3         0.5 1.00     0.00
4         0.7 2.33     0.85
5         0.9 9.00     2.20
  • The downside of Logit is that it is harder to interpret
  • At least the sign (postive/negative) tell us if probability goes up or down

Comparison: Linear regression \(Y = \beta_0 + \beta_1 X\)

Making Classifications

  • Logistic regression outputs probabilities, but we need to assign classes (Yes/No)
  • Common approach: choose threshold = 0.5
  • If \(p > 0.5\), predict Yes (default), otherwise predict No
  • Creates a confusion matrix comparing predicted vs actual values
> Default$pred_logistic <- factor(ifelse(Default$prob_logistic>0.5,"Yes","No"), levels=c("No","Yes"))

Threshold Selection

  • The default 0.5 threshold may not be optimal
  • Lower thresholds → more positives (higher recall, lower precision)
  • Higher thresholds → fewer positives (higher precision, lower recall)
  • Choice depends on the cost of errors:
    • Bank loans: better to avoid false negatives (catch more defaults)
    • Medical tests: better to avoid false positives (fewer unnecessary treatments)

ROC Curve Concept

  • ROC curve: plots True Positive Rate (Recall) vs False Positive Rate across thresholds
  • Helps visualize model’s ability to separate classes
  • A model with no predictive power lies on the diagonal (random guessing)
  • The closer the curve hugs the top-left, the better the model

Key Takeaways

Key Takeaways from Logistic Regression:

  • Probabilities over Classes: Logistic regression predicts probabilities, not just Yes/No outcomes.
  • Interpretability: Coefficients can be interpreted as odds ratios, showing the effect of predictors on the odds of the outcome.
  • Threshold Matters: Choice of probability threshold impacts precision, recall, and specificity.
  • Performance Evaluation: Use metrics like confusion matrix, ROC curve, and AUC to assess model performance.
  • Flexibility: Multiple predictors allow for more nuanced decision boundaries compared to simple rules.
  • Visualization Helps: Plotting probabilities and decision boundaries clarifies model behavior and trade-offs.
