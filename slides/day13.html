  • Our first highly flexible approach

What Makes k-NN Different?

Traditional Approach (Parametric):

  1. Assume a functional form (linear, logistic)
  2. Estimate parameters through optimization (least squares, MLE)
  3. Predict using learned parameters

k-NN Approach (Non-Parametric):

  1. Store all training data
  2. Find k nearest neighbors to new point
  3. Average their responses (regression) or vote (classification)

No optimization step! No parameters to estimate!

k-nearest neighbor classifier

  • Completely different way of approaching the problem
  • Idea: We want to predict \(y^*\) with characteristics \(x^*_1, ..., x^*_p\) where \(*\) means testing data
  1. Find the \(k\) cases in training data that are “closest” to the new cases characteristics and use their responses (\(y\)) to predict \(y^*\)
  2. Use responses of the “closest” cases to predict \(y^*\) using majority vote (most common response)
  • Distance measure: how to find closest cases in a p-dimensional predictor space?
    • if \(x\)s all quantitative: Euclidean distance
    • \(\sqrt{(x_{i1} - x_{j1})^2 + (x_{i2} - x_{j2})^2 + \cdots + (x_{ip} - x_{jp})^2}\)

k-nearest neighbor classifier

  • lazy-learner
    • no formal model
    • no pre-processing of the training data (just need training data to predict test responses)
  • Cons:
    • computationally intensive
    • accuracy depends on geometry of predictor space
    • how many neighbors \(k\) to pick?
      • what happens when \(k=1\)? \(k=n\)?
      • smaller \(k\): more overfitting (higher variance)
      • larger \(k\): less overfitting but more bias

Scaling is Critical!

  • the Euclidean distance is a function of variable scale
    • Just plotting the data is misleading because axis scales may be different
  • Methods relying on distance can be sensitive (i.e.not invariant) to the scale of the predictors
    • changing x2 from kilometers (x_df) to meters (x_df2) changes the case that is closest to test case 1
    • The variables with larger scale will get more weight when computing distance
  • With k-nn:
    • should scale (subtract mean divide by sd) predictors if they vary greatly in scale/variation
  • Other classifiers (e.g trees, logistic regression) are invariant to linear transformations of predictors

The Problem: When x2 is 1000x larger than x1:

  • x2 differences dominate the Euclidean distance calculation
  • x1 becomes essentially irrelevant
  • k-NN finds “nearest” neighbors based almost entirely on x2
  • The circular pattern (which depends on BOTH x1 and x2) is lost!

Comparing k-NN to Logistic Regression

Logistic Regression Accuracy: 0.7 
k-NN Accuracy (k=10): 0.787

Fundamental Difference:

  • Logistic Regression: Assumes linear decision boundary - struggles with circular patterns!
  • k-NN: No assumptions - can capture complex circular/ring patterns naturally

Comparing k-NN to Logistic Regression

               Method Accuracy Precision Recall Specificity F1_Score

2         k-NN (k=10)    0.787     0.640  0.667       0.839    0.653
  • k-NN dominates across all metrics for this non-linear pattern
  • Precision difference: k-NN has 0.382 higher precision
  • Recall difference: k-NN has 0.334 higher recall
  • F1-Score: k-NNs balanced performance shows 0.362 improvement

Why k-NN Excels Here:

  • Circular/ring pattern: Logistic regressions linear boundary cant capture the true decision boundary
  • Local adaptation: k-NN adapts to local patterns rather than forcing global linearity
  • No parametric bias: k-NN doesnt assume a specific functional form

When Logistic Regression Might Win:

  • Linear or near-linear decision boundaries
  • Limited training data (k-NN needs sufficient neighbors)
  • Need for interpretable coefficients
  • High-dimensional sparse data

Summary: k-NN as Our First Flexible Model

Revolutionary Concept:

  • No explicit optimization - just similarity search
  • No parametric assumptions - let data speak for itself
  • Highly flexible - can capture complex patterns

Key Takeaways:

  1. k-NN breaks from parametric tradition - no s to estimate!
  2. Flexibility controlled by k, not through parameter optimization
  3. Scaling predictors is critical for good performance
  4. Decision boundaries can be much more complex than linear
