Unsupervised Learning Recap

  • So far weve covered two unsupervised learning methods:
    • K-means clustering: partition data into K groups
    • Hierarchical clustering: create a dendrogram showing case similarity
  • Both methods group cases based on their X-variable values
  • Today: Principal Component Analysis (PCA)
    • transforms and reduces the variables themselves
    • finds the most important directions of variation in the data

The Dimensionality Problem

  • Modern datasets often have many variables (high-dimensional)
    • Customer data: age, income, spending across 100+ product categories
    • Gene expression: measurements on 10,000+ genes
    • Image data: thousands of pixel values
  • Problems with high dimensions:
    • Difficult to visualize
    • Models become computationally expensive
    • More susceptible to overfitting
  • Solution: reduce dimensions while retaining most information

What is Principal Component Analysis?

  • PCA finds new variables (principal components) that are:
    • linear combinations of the original variables
    • uncorrelated with each other
    • ordered by how much variation they explain
  • The first principal component (PC1) explains the most variation
  • The second principal component (PC2) explains the second most variation
  • And so on
  • We typically use only the first few PCs (dimension reduction!)

Simple 2D Example

  • Notice: SAT Math and Verbal scores are positively correlated
  • Can we capture this pattern with a single new variable?

Principal Component 1 (PC1)

  • PC1 is the line that captures the most variation in the data
  • Its the direction along which the data varies the most
  • Interpretation: PC1 is roughly like an “overall SAT score” ( average of Math and Verbal)

Principal Component 2 (PC2)

  • PC2 is perpendicular (orthogonal) to PC1
  • It captures the remaining variation not explained by PC1
  • PC1 and PC2 are uncorrelated
  • Interpretation: PC2 is roughly like the “difference” between Math and Verbal

PCA as Dimension Reduction

  • Original data: 2 dimensions (SATM, SATV)
  • After PCA: 2 principal components (PC1, PC2)
  • Key insight: If variables are correlated, most information is in PC1
    • We can often drop PC2 with little loss of information
    • Result: 2D 1D dimension reduction!
  • With \(p\) variables, we often use just the first 2-3 PCs
    • Makes visualization possible
    • Speeds up downstream analyses (clustering, modeling)

Mathematical Definition

  • For a dataset with p variables: \(X_1, X_2, ..., X_p\)
  • The first principal component is: \[ Z_1 = \phi_{11}X_1 + \phi_{21}X_2 + ... + \phi_{p1}X_p \]
  • Where \(\phi_1 = (\phi_{11}, \phi_{21},...,\phi_{p1})^T\) is the loading vector
  • Constraint: \(\sum_{j=1}^p \phi_{j1}^2 = 1\) (unit length)
  • \(\phi_1\) is chosen to maximize the variance of \(Z_1\)

Subsequent Principal Components

  • The second principal component \(Z_2\) is also a linear combination: \[ Z_2 = \phi_{12}X_1 + \phi_{22}X_2 + ... + \phi_{p2}X_p \]
  • Constraints:
    • Uncorrelated with \(Z_1\) (perpendicular loading vectors)
    • Maximizes variance subject to being uncorrelated with \(Z_1\)
  • This process continues for all p components
  • In practice: we use only the first M components (M << p)

Proportion of Variance Explained

  • Each PC explains a certain percentage of total variation
  • Proportion of Variance Explained (PVE) by PC m: \[ \text{PVE}_m = \frac{\text{Var}(Z_m)}{\sum_{k=1}^p \text{Var}(Z_k)} \]
  • Total PVE by first M components: \[ \sum_{m=1}^M \text{PVE}_m \]
  • Typically want first few PCs to explain 70-90% of variation

Important Notes on PCA

  • Always standardize your variables!
    • PCA is sensitive to the scale of variables
    • Use scale=TRUE in Rs prcomp() function
  • PCA is an unsupervised method
    • Does not use response variable y
    • Components are chosen to explain X variation, not y variation
  • PCA is useful for:
    • Data visualization (plot first 2-3 PCs)
    • Reducing computational burden
    • Addressing multicollinearity in regression

PCA in R: prcomp() function

  • Base R function: prcomp(x, center, scale)
    • x: matrix or data frame of numeric variables
    • center: should variables be centered? (default TRUE)
    • scale: should variables be standardized? (recommended TRUE)
  • Output: pca_out <- prcomp(...)
    • pca_out$x: the principal component scores (values of \(Z_1, Z_2, ...\))
    • pca_out$rotation: the loading vectors (\(\phi\) values)
    • pca_out$sdev: standard deviation of each PC
    • Use summary(pca_out) to see proportion of variance explained

When to Use PCA

  • Use PCA when:
    • You have many correlated variables
    • You want to visualize high-dimensional data
    • You need to speed up computation
    • You want to address multicollinearity
  • Be cautious when:
    • Variables are already uncorrelated (PCA wont help much)
    • You need to interpret individual variables (PCs are combinations)
    • You have categorical variables (PCA requires numeric data)

Summary

  • PCA transforms correlated variables into uncorrelated principal components
  • Components are ordered by variance explained
  • How to interpret PCs:
    • Look at loadings: which original variables have large weights?
    • PC1 captures the main pattern
    • PC2 captures the second pattern
    • Later PCs capture progressively smaller patterns
  • Use scree plots to decide how many PCs to retain
  • Always standardize variables before PCA
  • PCA is useful for:
    • Visualization (plot first 2-3 PCs)
    • Dimension reduction before modeling/clustering
    • Understanding structure in high-dimensional data
  • In R: use prcomp(data, scale=TRUE)
