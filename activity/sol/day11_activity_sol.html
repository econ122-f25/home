3. Why these metrics are useful

F1 score balances precision and recall, providing a single metric that accounts for both false positives and false negatives. This prevents the classifier from achieving high scores by simply predicting the majority class.

Balanced accuracy gives equal weight to sensitivity and specificity, ensuring both classes are considered equally important regardless of their prevalence in the dataset. This is crucial when the minority class (diabetes) is the one we care most about detecting.

Part 7 Reflection

1. Why accuracy is misleading

Accuracy is misleading because it can be high simply by predicting the majority class (no diabetes). A classifier that never detects diabetes achieves 65% accuracy but is clinically useless. With imbalanced data, accuracy doesnt reflect the classifiers ability to detect the important minority class.

2. When to prioritize precision vs recall

Prioritize Precision when:

  • False positives are costly (e.g., spam detection - marking important emails as spam)
  • Follow-up testing is expensive or invasive
  • Want to be confident in positive predictions

Prioritize Recall when:

  • False negatives are dangerous (e.g., cancer screening - missing a case could be fatal)
  • Early detection is critical
  • Follow-up testing is relatively inexpensive
  • In diabetes screening, recall is often more important to catch all cases

3. Improving rule-based classifier

  • Test different threshold combinations systematically
  • Add more predictor variables (insulin, BMI, family history)
  • Use AND logic for stricter screening or OR for broader screening
  • Create tiered rules (e.g., if glucose very high, predict pos regardless of age)
  • Weight predictors differently based on clinical importance

4. Extending with more predictors

With additional predictors, I would: - Create compound rules: (glucose > 125) OR (age > 50 AND insulin > 150) - Add thresholds for BMI (mass), blood pressure - Create risk score system where each factor adds points - Use nested if-else logic for more nuanced decisions - Consider interactions (e.g., high glucose + high BMI = higher risk)

# It's difficult to achieve both high sensitivity and specificity because:
# - Sensitivity requires low thresholds (predict pos often) to catch all positives
# - Specificity requires high thresholds (predict pos rarely) to avoid false positives
# - These goals are inherently conflicting
# - In imbalanced data, loose criteria needed for high sensitivity will
#   naturally classify many negatives as positive, reducing specificity
# - This fundamental trade-off is visualized in ROC curves

  pred <- factor(

           prob = c(1 - prop_pos, prop_pos)),

  )
  
  cm <- table(Predicted = pred, Actual = df$diabetes)
  TN <- cm[1,1]; FP <- cm[2,1]; FN <- cm[1,2]; TP <- cm[2,2]
  
  acc <- (TP + TN) / sum(cm)
  rec <- TP / (TP + FN)
  prec <- TP / (TP + FP)
  spec <- TN / (TN + FP)
  f1 <- 2 * (prec * rec) / (prec + rec)
  bal_acc <- (rec + spec) / 2
  
  random_results$accuracy[i] <- acc
  random_results$balanced_acc[i] <- bal_acc
  random_results$f1_score[i] <- f1
}

Key Takeaways:

  1. Accuracy is misleading with imbalanced data - the baseline achieves 65% by never detecting diabetes
  2. Balanced accuracy and F1 provide better evaluation for imbalanced classes
  3. Trade-offs are inevitable - improving recall often reduces precision
  4. Simple rules can be effective - glucose > 125 OR age > 50 performs reasonably well
  5. Cost considerations matter - in medical screening, false negatives are typically more costly than false positives
  6. Multiple predictors help - adding BMI improves detection
  7. Random guessing is bad - performs around 50% balanced accuracy, worse than informed rules
