Reflection and Analysis (5 minutes)

Based on your results, answer these questions:

  1. Pattern Dependency: Which patterns favor k-NN vs logistic regression? Why?

    Answer:

    • Linear patterns favor logistic regression (data_a results show this)
    • Non-linear patterns (circular, XOR) favor k-NN
    • k-NN can capture complex, non-parametric boundaries while logistic regression assumes linear relationships

  2. Flexibility Trade-off: What do you notice about k-NN’s ability to capture complex boundaries vs logistic regression’s simplicity?

    Answer:

    • k-NN creates more flexible, jagged decision boundaries that follow local patterns
    • Logistic regression creates smooth, linear boundaries
    • k-NN can overfit with complex boundaries, while logistic regression might underfit complex patterns

  3. Practical Considerations:

    • When would you choose k-NN over logistic regression in practice?
    • What are the computational trade-offs?

    Answer:

    • Choose k-NN when: relationships are clearly non-linear, you have sufficient data, interpretability is less important
    • Choose logistic regression when: you need interpretable coefficients, have limited data, relationships seem linear
    • Computational trade-offs: k-NN is slow at prediction time (must compute distances), logistic regression is fast once trained

  4. Scaling Impact: Calculate the percentage improvement from scaling in Part 2. Why was it so dramatic?

    Answer:

    ## [1] "Scaling improved accuracy by 18.5 percentage points"
    ## [1] "This is dramatic because income (in cents) completely dominated the distance calculation"
    ## [1] "Without scaling, test scores became irrelevant despite being predictive"

  5. Model Selection Challenge: If you had to choose ONE method for each pattern type (linear, circular, XOR), what would you choose and why?

    Answer:

    • Linear pattern: Logistic Regression (simpler, interpretable, designed for linear boundaries)
    • Circular pattern: k-NN (naturally captures radial patterns)
    • XOR pattern: k-NN (logistic regression cannot handle XOR without feature engineering)

Key Takeaways

  1. k-NN is sensitive to the choice of k - too small causes overfitting, too large causes underfitting
  2. Scaling is absolutely critical for k-NN - features on different scales will dominate distance calculations
  3. Pattern complexity matters - k-NN excels with non-linear patterns, logistic regression with linear relationships
  4. Computational trade-offs - k-NN is expensive at prediction time, logistic regression after training is fast
  5. Dimensionality curse - k-NN struggles as dimensionality increases relative to sample size

You’ve now experienced the power and limitations of k-NN compared to parametric methods. Understanding when each method excels is crucial for real-world machine learning success!
